# Защита API Backend

Краткий чеклист и настройки безопасности.

## Что уже включено

- **JWT** — авторизация по токену; защищённые маршруты требуют `Authorization: Bearer <token>`.
- **Роли** — `owner`, `client`, `admin`; доступ к маршрутам по ролям (`requireRole`, `adminMiddleware`).
- **Helmet** — безопасные HTTP-заголовки (X-Content-Type-Options, X-Frame-Options и др.).
- **Лимит на логин/регистрацию** — не более 10 попыток с одного IP за 15 минут (`authLimiter`).
- **Общий лимит запросов** — не более 300 запросов с одного IP за 15 минут (`apiLimiter`). Порог задаётся в `.env`: `RATE_LIMIT_MAX_PER_15MIN`.
- **Trust proxy** — учёт реального IP за Nginx (`X-Forwarded-For`).

## Что настроить в продакшене

### 1. HTTPS

API должен отдаваться только по HTTPS (Nginx + Let's Encrypt). В Nginx — редирект HTTP → HTTPS.

### 2. CORS

По умолчанию разрешены запросы с любых доменов. Чтобы разрешить только свои приложения, в `.env` укажите:

```env
ALLOWED_ORIGINS=https://api.onthewater.ru,https://onthewater.ru,capacitor://localhost
```

Через запятую, без пробелов внутри URL. Для мобильного приложения (Expo/Capacitor) при необходимости добавьте `capacitor://localhost` или свой scheme.

### 3. Секреты

- **JWT_SECRET** — длинная случайная строка (например 32+ символа), хранить только в `.env`, не коммитить.
- **DATABASE_URL** / пароли БД — только в `.env`.
- Файл `.env` не должен попадать в Git (уже в `.gitignore`).

### 4. Лимит запросов

В `.env` можно задать свой порог:

```env
RATE_LIMIT_MAX_PER_15MIN=200
```

При необходимости можно ослабить или усилить лимит.

### 5. Nginx

- Ограничение размера тела запроса уже учтено в конфиге (`client_max_body_size 11M`).
- Не отдавать наружу порт 3000 — только Nginx на 80/443.

### 6. База данных

- Подключение к PostgreSQL только с локального хоста или по защищённому каналу.
- Пользователь приложения — с минимальными правами (только нужные таблицы/операции).

### 7. Обновления

Регулярно обновлять зависимости: `npm audit`, `npm update`, обновление ОС на сервере.
