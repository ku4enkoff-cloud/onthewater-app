# BoatRent

Мобильное приложение для аренды катеров и яхт (клиенты и владельцы). React Native (Expo) + опциональный backend на Node.js.

## Структура проекта

- **mobile/** — приложение Expo (React Native), два варианта из одного кода: клиент и владелец
- **admin/** — веб-сайт для администрирования (клиентское и владельческое приложения): пользователи, катера, бронирования
- **backend/** или **server/** — бэкенд API (Express), опционально для локальной разработки

В **mobile** общий код вынесен в **src/shared/** (тема, API, контекст авторизации), отдельно — **src/auth/** (вход/регистрация), **src/client/** (приложение для клиентов), **src/owner/** (приложение для владельцев). Точка входа выбирается переменной `EXPO_PUBLIC_APP_VARIANT` (`client` по умолчанию, `owner` — приложение владельца).

## Запуск

### Мобильное приложение

```bash
cd mobile
npm install
npm start
```

Команда `npm start` явно запускает **приложение для клиентов** (главная с катерами). Если после переключения с owner всё ещё открывается экран владельца — закройте сервер (Ctrl+C), затем: `npm run start:client:clear`.

**Приложение для владельцев:** сначала установите зависимость и запустите сервер с вариантом `owner`:

```bash
cd mobile
npm install
npm run start:owner
```

**Как открыть приложение в Expo Go:**

1. На компьютере в терминале выполните (из папки проекта):
   ```bash
   cd mobile
   npm install
   npx expo start
   ```
2. Дождитесь появления в терминале QR-кода и строки вида `exp://192.168.x.x:8081`.
3. На телефоне в **Expo Go** нажмите **«Enter URL manually»** (Ввести URL вручную).
4. Введите адрес из терминала, например: `exp://192.168.1.100:8081` (подставьте свой IP и порт).
5. Либо отсканируйте QR-код из терминала камерой телефона (если Expo Go умеет открывать ссылку).

**Важно:** телефон и компьютер должны быть в одной Wi‑Fi сети. Если не подключается — запустите с туннелем: `npx expo start --tunnel` (потребуется установка @expo/ngrok при первом запуске).

Если раньше запускали приложение владельца — лучше очистить кэш: `npm run start:owner:clear`. Если меняли картинки на главном экране — перезапустите с очисткой кэша: `npm run start:client:clear`.

Для сборки под Android: `npx expo run:android`. Для iOS: `npx expo run:ios`.

Адрес API задаётся в `mobile/src/shared/infrastructure/config.js` (по умолчанию `http://localhost:3000`, для Android-эмулятора — `http://10.0.2.2:3000`).

### Бэкенд (опционально)

```bash
cd server
npm install
npm start
```

Сервер будет доступен на `http://localhost:3000`.

**Тестовые аккаунты:**

- Клиент: `client@test.ru` / `123`
- Владелец: `owner@test.ru` / `123`
- Админ (веб-панель): `admin@test.ru` / `123`

### Веб-админка

Панель для управления пользователями, катерами и бронированиями. Работает с тем же API (server).

```bash
cd admin
npm install
npm run dev
```

Откройте в браузере: **http://localhost:5174**. Войдите под `admin@test.ru` / `123`. Доступны разделы: дашборд, пользователи, катера (с изменением статуса: активен / на модерации / заблокирован), бронирования. Перед запуском админки должен быть запущен сервер (`cd server && npm start`).

## API (описание для бэкенда)

Базовый URL: `http://localhost:3000` (или переменная окружения в приложении).

### Авторизация

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/auth/login` | Body: `{ email, password }`. Ответ: `{ token, user }`. |
| POST | `/auth/register` | Body: `{ email, password, name? }`. Ответ: `{ token, user }`. |
| GET | `/auth/me` | Заголовок `Authorization: Bearer <token>`. Ответ: объект пользователя `{ id, email, name, role, created_at }`. |

### Катера

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/boats` | Query: `lat`, `lng`, `radius` (опционально). Список катеров. |
| GET | `/boats/:id` | Один катер по id. |
| POST | `/boats` | Auth. Multipart: `title`, `description`, `type_id`, `year`, `length_m`, `capacity`, `location_city`, `location_address`, `lat`, `lng`, `price_per_hour`, `price_per_day`, `captain_included`, `has_captain_option`, `rules`, файлы `photos`. |

### Бронирования (клиент)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/bookings` | Auth. Список бронирований текущего пользователя. |
| GET | `/bookings/:id` | Auth. Детали бронирования. |
| POST | `/bookings/:id/cancel` | Auth. Отмена бронирования. |

### Бронирования (владелец)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/owner/bookings` | Auth. Список бронирований катеров владельца. |
| POST | `/owner/bookings/:id/confirm` | Auth. Подтвердить бронирование. |
| POST | `/owner/bookings/:id/decline` | Auth. Отклонить бронирование. |

### Чаты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/chats` | Auth. Список чатов пользователя. |
| GET | `/chats/:id` | Auth. Данные чата. |
| GET | `/chats/:id/messages` | Auth. Сообщения чата. |
| POST | `/chats/:id/messages` | Auth. Body: `{ text }`. Отправить сообщение. |
| GET | `/owner/chats` | Auth. Список чатов владельца. |

## Роли

- **client** — клиент: поиск катеров, бронирования, чаты с владельцами.
- **owner** — владелец: добавление катеров, управление бронированиями, чаты с клиентами.
- **admin** — администратор: вход только в веб-админку; просмотр и управление пользователями, катерами, бронированиями.

После входа приложение показывает соответствующие табы (клиент или владелец).

## Технологии

- **mobile:** Expo ~51, React Navigation, axios, react-native-yamap, AsyncStorage, lucide-react-native.
- **server:** Express, cors, multer (для загрузки фото).
